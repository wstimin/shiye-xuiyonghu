# 视野 3x-ui 用户管理系统 宝塔部署教程

本文只讲宝塔 Linux 面板部署。当前项目是新架构版本：NestJS API、Vue 管理端、Vue 用户端、MySQL、Prisma，不再使用旧版 `server.js` 和网页安装向导。

## 1. 准备条件

需要准备：

- 宝塔 Linux 面板
- Nginx 或 OpenResty
- MySQL 8.0+ 或 MariaDB 兼容版本
- Node.js 20+
- 一个域名，推荐使用 HTTPS
- 本项目完整源码

默认信息：

```text
项目目录：/www/wwwroot/shiye
API 端口：3388
用户端入口：https://你的域名/
管理端入口：https://你的域名/admin/
API 前缀：https://你的域名/api/
```

## 2. 选择部署方式和安装宝塔软件

如果你想不用面板手动上传源码，也可以在宝塔终端直接执行一键脚本。脚本会询问访问方式：选择跳过域名就是 `IP:3388` 访问；选择域名访问会继续输入域名，并可自动申请 HTTPS 证书。

一键脚本默认是精简运行目录：构建时只获取必要项目文件，安装完成后只保留运行必需项，不会把 README、部署文档、安装脚本、前后端源码和 `.git` 长期留在服务器运行目录。

```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh)"
```

一键脚本部署成功后，项目由 systemd 服务 `shiye-api` 管理，不需要再到宝塔里创建 Node 项目。服务器上可以随时打开管理菜单：

```bash
sudo shiye
```

菜单标题只显示“管理面板”，可执行安装/更新、查看状态、重启、日志、配置或取消域名、重新构建、数据库迁移、备份和卸载。

也可以提前带入域名和 HTTPS 参数：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | sudo env DOMAIN=panel.example.com ENABLE_NGINX=yes ENABLE_HTTPS=yes CERTBOT_EMAIL=admin@example.com bash
```

继续使用宝塔面板部署时，按下面步骤操作。

注意：一键脚本部署和下面的宝塔手动源码部署二选一即可。不要一边用一键脚本安装到 `/opt/shiye`，一边又在宝塔里用另一份 `/www/wwwroot/shiye` 源码创建 Node 项目，否则会出现两个运行目录、两个启动方式，排查会很乱。

进入宝塔：

```text
软件商店
```

确认安装：

```text
Nginx 或 OpenResty
MySQL 或 MariaDB
Node.js 版本管理器，安装 Node.js 20.x
PM2 管理器，可选
```

在宝塔终端确认版本：

```bash
node -v
npm -v
```

Node.js 必须是 20 或更高版本。

## 3. 创建数据库

进入宝塔：

```text
数据库 -> 添加数据库
```

推荐填写：

```text
数据库名：shiye_management
用户名：shiye
密码：自己生成一个强密码
访问权限：本地服务器
字符集：utf8mb4
```

记录数据库连接信息，稍后写入 `.env`：

```text
DATABASE_URL=mysql://shiye:你的数据库密码@127.0.0.1:3306/shiye_management
```

## 4. 上传项目

推荐目录：

```text
/www/wwwroot/shiye
```

上传并解压源码后，目录根部必须能看到：

```text
/www/wwwroot/shiye/package.json
/www/wwwroot/shiye/apps/
/www/wwwroot/shiye/packages/
/www/wwwroot/shiye/prisma/
/www/wwwroot/shiye/infra/
/www/wwwroot/shiye/install.sh
```

不要解压成多一层目录，例如：

```text
/www/wwwroot/shiye/shiye-xuiyonghu/package.json
```

如果多了一层，把里面的所有文件移动到 `/www/wwwroot/shiye`。

## 5. 配置环境变量

在宝塔终端执行：

```bash
cd /www/wwwroot/shiye
cp .env.example .env
```

编辑 `.env`，至少修改：

```env
NODE_ENV=production
PORT=3388
PUBLIC_WEB_URL=https://你的域名
DATABASE_URL=mysql://shiye:你的数据库密码@127.0.0.1:3306/shiye_management
SESSION_SECRET=换成强随机字符串
ENCRYPTION_KEY=换成32字节base64密钥
CARD_HASH_SECRET=换成强随机字符串
DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=换成强后台密码
```

生成密钥：

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

`ENCRYPTION_KEY` 必须备份好，丢失后已保存的 3x-ui Token、密码和支付密钥无法解密。

## 6. 安装依赖、初始化数据库和构建

在宝塔终端执行：

```bash
cd /www/wwwroot/shiye
npm ci
npm run install:prod
npm prune --omit=dev
```

注意：不要直接执行 `npm ci --omit=dev` 或 `npm install --omit=dev` 后再构建。前端和 TypeScript 构建需要 devDependencies。正确顺序是先完整安装依赖，构建完成后再 `npm prune --omit=dev`。

如果只是更新数据库和构建，仍然推荐执行：

```bash
npm ci
npm run install:prod
npm prune --omit=dev
```

## 7. 创建 Node 项目

进入宝塔：

```text
网站 -> Node 项目
```

推荐填写：

```text
项目名称：shiye-api
项目目录：/www/wwwroot/shiye
启动命令：npm start
Node 版本：20.x
端口：3388
运行用户：www，或宝塔默认用户
```

如果宝塔要求填写环境变量，可以留空，因为项目会读取 `/www/wwwroot/shiye/.env`。

启动后在服务器终端测试：

```bash
curl http://127.0.0.1:3388/api/health
curl http://127.0.0.1:3388/api/setup/status
```

能返回 JSON 即表示 API 正常。

## 8. 配置网站和反向代理

进入宝塔：

```text
网站 -> 添加站点
```

站点根目录可以指向：

```text
/www/wwwroot/shiye/dist/user-web
```

创建后进入站点配置，添加或修改 Nginx 配置。推荐使用下面的完整配置，把 `你的域名` 和路径按实际情况替换：

```nginx
server {
    listen 80;
    server_name 你的域名;

    root /www/wwwroot/shiye/dist/user-web;
    index index.html;

    location = /api {
        return 301 /api/;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3388/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location = /admin {
        return 301 /admin/;
    }

    location /admin/assets/ {
        alias /www/wwwroot/shiye/dist/admin-web/assets/;
        try_files $uri =404;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location /admin/ {
        alias /www/wwwroot/shiye/dist/admin-web/;
        try_files $uri $uri/ /admin/index.html;
    }

    location /assets/ {
        try_files $uri =404;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

保存后测试并重载 Nginx。

如果使用宝塔的“反向代理”功能只代理到 `http://127.0.0.1:3388`，前端静态页面不会自动由 API 提供。当前架构推荐按上面的 Nginx 配置直接托管 `dist/user-web` 和 `dist/admin-web`。

## 9. 申请 SSL 证书

进入宝塔站点设置：

```text
网站 -> 选择站点 -> SSL
```

申请前确认：

```text
域名已经解析到服务器 IP
宝塔安全和云服务器安全组放行 80/443
http://你的域名 可以正常访问
```

证书成功后，建议开启强制 HTTPS，并把 `.env` 里的 `PUBLIC_WEB_URL` 设置为 `https://你的域名`。

## 10. 支付回调地址

本项目只有用户充值余额业务。用户可输入任意金额，支付成功后增加余额，再用余额续费节点。

支付平台回调地址格式：

```text
https://你的域名/api/payments/epay/notify
https://你的域名/api/payments/bepusdt/notify
https://你的域名/api/payments/alipay/notify
https://你的域名/api/payments/wechat/notify
```

支付结果页：

```text
https://你的域名/payment/result?trade_no=订单号
```

## 11. 更新项目

如果使用一键脚本默认精简部署，运行目录不是 Git 仓库，不能直接 `git pull`。推荐重新执行一键脚本，脚本会保留现有 `.env`，重新获取最新项目文件、迁移数据库、构建并重启服务：

```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh)"
```

也可以执行 `sudo shiye`，在“管理面板”菜单里选择：

```text
1. 安装/更新项目
```

只有手动上传完整源码时，才使用下面的源码更新方式。

更新前备份：

```text
MySQL 数据库 shiye_management
/www/wwwroot/shiye/.env
自定义 Nginx 配置
```

更新命令：

```bash
cd /www/wwwroot/shiye
git pull
npm ci
npm run install:prod
npm prune --omit=dev
```

然后在宝塔里重启 Node 项目。

## 12. 卸载项目

如果使用一键脚本部署，推荐执行：

```bash
sudo shiye
```

然后选择：

```text
10. 卸载项目
11. 卸载项目并删除数据库
```

也可以直接使用命令。默认卸载程序、systemd 服务和脚本创建的 Nginx 配置，保留数据库：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/uninstall.sh | sudo bash
```

彻底卸载并删除默认数据库 `shiye_management` 和默认数据库用户 `shiye`：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/uninstall.sh | sudo env DELETE_DATABASE=yes bash
```

如果是宝塔手动源码部署，需要在宝塔里停止并删除 Node 项目、删除对应站点或反向代理配置，再按需删除 `/www/wwwroot/shiye` 和数据库。删除数据库会清空用户、余额、订单、节点、卡密、支付配置和同步日志，操作前务必备份 `.env` 和 MySQL 数据。

## 13. 常见问题

### 启动失败，提示找不到 package.json

项目目录填错，或源码多解压了一层。确认运行目录下直接存在：

```text
package.json
apps/
packages/
prisma/
```

### API 正常，但页面 404

检查 Nginx `root` 是否指向 `/www/wwwroot/shiye/dist/user-web`，并确认已经执行 `npm run install:prod` 生成 `dist`。

### 管理端资源 404

检查 `/admin/assets/` 是否使用 `alias /www/wwwroot/shiye/dist/admin-web/assets/;`，不要代理到 API。

### 数据库连接失败

检查 `.env` 中的 `DATABASE_URL`。宝塔本机 MySQL 通常是：

```text
127.0.0.1:3306
```

### 支付回调不到账

检查：

- `.env` 的 `PUBLIC_WEB_URL` 是否是公网 HTTPS 域名
- 支付平台回调地址是否是 `/api/payments/:provider/notify`
- Nginx 是否正确把 `/api/` 代理到 `127.0.0.1:3388/api/`
- 后台支付通道密钥和签名方式是否正确

## 14. 备份建议

至少备份：

```text
MySQL 数据库 shiye_management
/www/wwwroot/shiye/.env
Nginx 站点配置
```

`.env` 里的 `ENCRYPTION_KEY` 必须保留，否则加密字段无法恢复。
