# 十夜管理系统 宝塔部署教程

本文只讲宝塔面板部署，不使用一键安装脚本。适合已经安装宝塔 Linux 面板，并希望通过宝塔的网站、MySQL、Node.js 项目管理器来部署十夜管理系统的用户。

项目基于 3-xui 面板 3.4.1 版本开发和测试。部署完成后，请先在管理后台测试 3-xui 节点连接，再创建用户并同步。

## 1. 准备条件

需要准备：

- 宝塔 Linux 面板
- Nginx 或 OpenResty
- MySQL 5.7/8.0 或 MariaDB
- Node.js 版本管理器，建议 Node.js 20
- 一个域名，可选；没有域名也可以用 `http://服务器IP:3388` 访问
- 本项目完整源码

默认信息：

```text
项目端口：3388
用户入口：http://服务器IP:3388/
管理员入口：http://服务器IP:3388/admin
默认管理员账号：admin
默认管理员密码：admin123
```

公网使用建议登录后修改默认管理员密码。

## 2. 安装宝塔软件

进入宝塔：

```text
软件商店
```

确认安装：

```text
Nginx 或 OpenResty
MySQL 或 MariaDB
Node.js 版本管理器
PM2 管理器，可选
```

在 Node.js 版本管理器中安装 Node.js 20.x。安装后可以在服务器终端确认：

```bash
node -v
npm -v
```

## 3. 创建数据库

进入宝塔：

```text
数据库 -> 添加数据库
```

推荐填写：

```text
数据库名：shiye_management
用户名：shiye
密码：自己生成一个强密码
访问权限：本地服务器
字符集：utf8mb4
```

把数据库名、用户名、密码记下来。首次打开十夜管理系统时，需要在网页安装向导里填写。

宝塔本机数据库一般这样填：

```text
数据库地址：127.0.0.1
数据库端口：3306
数据库名称：shiye_management
数据库账号：shiye
数据库密码：创建数据库时设置的密码
```

如果你的 MySQL 是 Docker 容器或远程数据库，请填写实际连接地址。

## 4. 上传项目文件

推荐项目目录：

```text
/www/wwwroot/shiye-management-system
```

进入宝塔：

```text
文件 -> /www/wwwroot -> 新建目录 shiye-management-system
```

上传项目压缩包并解压。解压完成后，目录结构必须类似：

```text
/www/wwwroot/shiye-management-system/package.json
/www/wwwroot/shiye-management-system/server.js
/www/wwwroot/shiye-management-system/public/
/www/wwwroot/shiye-management-system/README.md
```

注意：`package.json` 必须在项目运行目录根部。不要解压成下面这种多一层目录的结构：

```text
/www/wwwroot/shiye-management-system/3-xuiguanli-shangye/package.json
```

如果多了一层，把里面的所有文件移动到 `/www/wwwroot/shiye-management-system`。

## 5. 安装依赖

可以在宝塔终端执行：

```bash
cd /www/wwwroot/shiye-management-system
npm install --omit=dev
node --check server.js
node --check public/app.js
```

如果宝塔 Node 项目管理器支持“安装依赖命令”，填写：

```bash
npm install --omit=dev
```

## 6. 创建 Node.js 项目

进入宝塔：

```text
网站 -> Node 项目，或软件商店 -> Node 项目管理器
```

创建项目时推荐填写：

```text
项目名称：shiye-management-system
项目目录：/www/wwwroot/shiye-management-system
启动文件：server.js
启动命令：npm start
Node 版本：20.x
端口：3388
运行用户：www，或宝塔默认用户
```

如果只能填写启动命令，填写：

```bash
npm start
```

或：

```bash
node server.js
```

环境变量可以先不填。首次访问系统会显示网页安装向导，在网页里绑定 MySQL。

如果面板要求必须填写环境变量，可以只填写：

```text
PORT=3388
ADMIN_PATH=/admin
```

不要随便填写错误的 `MYSQL_HOST`、`MYSQL_USER` 等变量，否则可能导致安装向导不出现，或者一直连接失败。

## 7. 启动并测试

启动 Node.js 项目后，先用服务器本机测试：

```bash
curl -i http://127.0.0.1:3388/
```

如果返回 HTML 页面或首次安装页面，说明项目已经启动。

浏览器访问：

```text
http://服务器IP:3388/
```

如果访问不了，检查：

```text
1. 宝塔安全里是否放行 3388
2. 云服务器安全组是否放行 3388
3. Node 项目是否正在运行
4. 项目日志是否提示 package.json 或依赖错误
```

如果后续配置了 Nginx 反向代理，只需要对外开放 `80/443`，可以不开放 `3388`。

## 8. 首次网页安装向导

首次访问会进入“首次安装”页面，填写数据库：

```text
数据库地址：127.0.0.1
数据库端口：3306
数据库名称：shiye_management
数据库账号：shiye
数据库密码：宝塔创建数据库时设置的密码
管理员入口：/admin，或你想自定义的路径
```

点击连接并安装。安装成功后，系统会自动建表，并保存配置到：

```text
data/config.json
```

这个文件包含数据库连接信息，迁移和备份时需要保留，不要公开。

安装完成后访问：

```text
用户入口：http://服务器IP:3388/
管理员入口：http://服务器IP:3388/admin
```

默认管理员：

```text
账号：admin
密码：admin123
```

## 9. 创建网站和反向代理

如果你要用域名访问，进入宝塔：

```text
网站 -> 添加站点
```

填写你的域名，例如：

```text
gl.example.com
```

PHP 版本选择纯静态或不启用 PHP。创建完成后，进入站点设置：

```text
反向代理 -> 添加反向代理
```

推荐填写：

```text
代理名称：shiye-management-system
目标 URL：http://127.0.0.1:3388
发送域名：$host
```

如果宝塔需要手动配置 Nginx，示例：

```nginx
location / {
    proxy_pass http://127.0.0.1:3388;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

保存后访问：

```text
http://你的域名/
```

管理员入口：

```text
http://你的域名/admin
```

## 10. 申请 SSL 证书

进入宝塔站点设置：

```text
网站 -> 选择站点 -> SSL
```

可以选择：

```text
Let's Encrypt
```

申请前确认：

```text
域名已经解析到服务器 IP
宝塔安全和云服务器安全组放行 80/443
http://你的域名 可以正常访问
Cloudflare 小黄云建议先关闭，申请成功后再按需要开启
```

证书申请成功后，开启强制 HTTPS 可选。系统本身不强制 HTTPS，但公网使用更推荐开启。

## 11. 更新项目

更新前先备份：

```text
/www/wwwroot/shiye-management-system/data/
MySQL 数据库 shiye_management
```

更新步骤：

```text
1. 停止 Node.js 项目
2. 上传新版项目文件覆盖旧程序文件
3. 保留 data/config.json 和 data/.secret
4. 执行 npm install --omit=dev
5. 启动 Node.js 项目
6. 查看项目日志确认无报错
```

终端命令示例：

```bash
cd /www/wwwroot/shiye-management-system
npm install --omit=dev
node --check server.js
node --check public/app.js
```

## 12. 常见问题

### 启动失败，提示找不到 package.json

项目目录填错了，或解压后多了一层目录。确认运行目录下有：

```text
package.json
server.js
public/
```

### 访问 IP:3388 没反应

检查宝塔安全和云服务器安全组是否放行 `3388`。如果使用域名反代访问，可以只放行 `80/443`。

### 数据库连接失败

宝塔本机 MySQL 通常使用：

```text
127.0.0.1:3306
```

如果 MySQL 是 Docker 或远程数据库，需要填写实际地址，并确认数据库账号允许从当前服务器连接。

### 首次安装页面没有出现

检查是否已经存在：

```text
data/config.json
```

如果是全新安装但页面没出现，检查 Node 项目环境变量里是否写了错误的 MySQL 配置。

### 管理员和用户不能同浏览器同时登录

同一个浏览器会共享 Cookie。测试时建议管理员用一个浏览器，用户用另一个浏览器或无痕窗口。

## 13. 备份建议

至少备份：

```text
MySQL 数据库 shiye_management
/www/wwwroot/shiye-management-system/data/config.json
/www/wwwroot/shiye-management-system/data/.secret
```

如果丢失 `data/.secret`，之前保存的 3-xui Token、SOCKS 密码等敏感信息可能无法解密。
