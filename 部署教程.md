# 十夜管理系统部署教程

本教程适合把十夜管理系统部署到 Linux 服务器。系统可以直接使用 HTTP 访问，不强制配置 Nginx 或 HTTPS；如果你会配置反向代理，也可以后续再加。

## 1. 最快安装方式

服务器执行下面命令，把地址换成你的仓库：

```bash
REPO_URL=https://github.com/你的用户名/你的仓库.git bash <(curl -fsSL https://raw.githubusercontent.com/你的用户名/你的仓库/main/install.sh)
```

注意：发布到自己的 GitHub 后，把 `你的用户名/你的仓库` 改成真实仓库地址。如果默认分支不是 `main`，也要把链接里的 `main` 改成实际分支名。

安装完成后访问：

```text
http://服务器IP:3388
```

默认登录信息：

```text
账号：admin
密码：admin123
```

系统不会强制你修改密码，但公网部署建议进入右上角“账号安全”修改管理员账号和密码。

## 2. 一键脚本做了什么

`install.sh` 会自动完成：

- 检查或安装 Node.js 20
- 安装 Git
- 把项目放到 `/opt/shiye-management-system`
- 创建 `data/` 数据目录
- 检查 `server.js` 和 `public/app.js` 语法
- 创建 systemd 服务
- 启动服务并设置开机自启

默认服务名：

```text
shiye-management-system
```

默认安装路径：

```text
/opt/shiye-management-system
```

默认端口：

```text
3388
```

## 3. 自定义安装路径或端口

自定义端口：

```bash
PORT=8080 REPO_URL=https://github.com/你的用户名/你的仓库.git bash <(curl -fsSL https://raw.githubusercontent.com/你的用户名/你的仓库/main/install.sh)
```

自定义安装目录：

```bash
APP_DIR=/opt/my-shiye-panel REPO_URL=https://github.com/你的用户名/你的仓库.git bash <(curl -fsSL https://raw.githubusercontent.com/你的用户名/你的仓库/main/install.sh)
```

指定 GitHub 仓库：

```bash
REPO_URL=https://github.com/你的用户名/你的仓库.git bash install.sh
```

## 4. 常用运行命令

查看服务状态：

```bash
systemctl status shiye-management-system
```

重启服务：

```bash
systemctl restart shiye-management-system
```

停止服务：

```bash
systemctl stop shiye-management-system
```

启动服务：

```bash
systemctl start shiye-management-system
```

查看实时日志：

```bash
journalctl -u shiye-management-system -f
```

查看最近日志：

```bash
journalctl -u shiye-management-system -n 100 --no-pager
```

检查端口是否监听：

```bash
ss -lntp | grep 3388
```

本机测试访问：

```bash
curl -i http://127.0.0.1:3388/
```

## 5. 手动安装方式

如果不使用一键脚本，可以手动安装。

安装 Node.js 20：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs git
node -v
```

下载项目：

```bash
cd /opt
git clone https://github.com/你的用户名/你的仓库.git shiye-management-system
cd /opt/shiye-management-system
```

语法检查：

```bash
node --check server.js
node --check public/app.js
```

临时启动：

```bash
PORT=3388 node server.js
```

后台启动：

```bash
cd /opt/shiye-management-system
pkill -f "node server.js"
nohup node server.js > app.log 2>&1 &
tail -n 50 app.log
```

## 6. 手动创建 systemd 服务

创建服务文件：

```bash
nano /etc/systemd/system/shiye-management-system.service
```

写入：

```ini
[Unit]
Description=Shiye Management System
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/shiye-management-system
Environment=PORT=3388
ExecStart=/usr/bin/node server.js
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动：

```bash
systemctl daemon-reload
systemctl enable --now shiye-management-system
systemctl status shiye-management-system
```

## 7. 配置 3-xui 节点

进入“3x-ui 节点”页面，点击“添加 3x-ui 节点”。

如果你的 3-xui 地址是：

```text
http://example.com:2053/
```

填写：

```text
协议：http
地址：example.com
端口：2053
基础路径：/
账号：可留空
密码：可留空
API Token：填写 3-xui 的 API Token
```

如果你的 3-xui 地址是：

```text
https://example.com:2053/custompath/
```

填写：

```text
协议：https
地址：example.com
端口：2053
基础路径：/custompath
账号：可留空
密码：可留空
API Token：填写 3-xui 的 API Token
```

添加后点击“测试”。能显示可用 Inbound ID 就说明连接成功。

## 8. 创建用户并同步

进入“用户管理”，点击“新建用户”。

已有 3-xui 入站时：

```text
选择 3x-ui 节点
填写 Inbound ID
Client Email 可留空自动生成
UUID 可留空自动生成
保存
点击同步
```

自动创建入站时：

```text
选择 3x-ui 节点
Inbound ID 留空
勾选自动创建入站
选择入站模板
端口可留空自动选择
保存
点击同步
```

自动创建入站会检查 3-xui 现有入站端口，避免端口冲突。

## 9. SOCKS 出站中转

进入“SOCKS 出站”，添加 SOCKS 节点。

然后编辑用户：

```text
启用 SOCKS 中转
选择 SOCKS 节点
保存
点击同步
```

同步时系统会写入 3-xui 的 Xray 模板：

- `outbounds` 添加 SOCKS 出站
- `routing.rules` 添加入站 tag 到 SOCKS tag 的规则
- 调用 Xray 重载接口

## 10. 更新项目

如果使用 Git 部署：

```bash
cd /opt/shiye-management-system
git pull
node --check server.js
node --check public/app.js
systemctl restart shiye-management-system
journalctl -u shiye-management-system -n 80 --no-pager
```

如果是手动上传覆盖文件：

```bash
cd /opt/shiye-management-system
node --check server.js
node --check public/app.js
systemctl restart shiye-management-system
```

## 11. 备份和恢复

需要备份两个文件：

```text
data/db.json
data/.secret
```

备份命令：

```bash
cd /opt/shiye-management-system
tar -czf /root/shiye-backup-$(date +%F).tar.gz data/db.json data/.secret
```

恢复命令：

```bash
cd /opt/shiye-management-system
tar -xzf /root/shiye-backup-日期.tar.gz
systemctl restart shiye-management-system
```

如果丢失 `data/.secret`，之前保存的 3-xui Token、密码、SOCKS 密码将无法解密。

## 12. 可选：Nginx 反向代理

不会配置 Nginx 可以跳过这一节，直接使用：

```text
http://服务器IP:3388
```

安装 Nginx：

```bash
apt install -y nginx
```

示例配置：

```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:3388;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

检查并重载：

```bash
nginx -t
systemctl reload nginx
```

## 13. 可选：HTTPS

如果你已经配置好域名和 Nginx，可以使用 Certbot：

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d example.com
```

HTTPS 不是系统运行的硬性要求，但公网使用时更推荐。

## 14. 安全建议

- 默认账号密码可以使用，但建议修改。
- 公网部署建议使用 HTTPS。
- 不要公开 `data/db.json` 和 `data/.secret`。
- 不要公开 3-xui API Token。
- 如果不使用 Nginx，安全组需要放行 `3388`。
- 如果使用 Nginx，建议只放行 `80/443`，不要放行 `3388`。
