# 十夜 3x-ui 用户管理系统 宝塔部署教程

本文只讲宝塔 Linux 面板部署。当前项目是新架构版本：NestJS API、Vue 管理端、Vue 用户端、MySQL、Prisma。

## 1. 准备条件

需要准备：

- 宝塔 Linux 面板
- Nginx 或 OpenResty
- MySQL 8.0+ 或 MariaDB 兼容版本
- Node.js 20+
- 一个域名，推荐使用 HTTPS
- 本项目 Linux 预构建部署包，或完整源码

默认信息：

```text
项目目录：/www/wwwroot/shiye
API 端口：3388
用户端入口：https://你的域名/
管理端入口：https://你的域名/admin/
API 前缀：https://你的域名/api/
```

## 2. 选择部署方式和安装宝塔软件

如果你想不用面板手动上传部署包，也可以在宝塔终端直接执行一键脚本。下面命令默认在 `root` 用户下执行；如果当前不是 `root`，请先切换到 `root` 用户。脚本默认优先下载 Linux 预构建包，包不可用时回退源码构建；安装为 `IP:3388` 访问，安装完成后可通过 `shiye` 菜单第 6 项配置域名、Nginx 和 HTTPS。

一键脚本默认是精简运行目录：优先使用预构建运行文件，安装完成后只保留运行必需项，不会把 README、部署文档、安装脚本、前后端源码和 `.git` 长期留在服务器运行目录。

默认构建包地址：`https://github.com/wstimin/shiye-3xuigl-L3/releases/latest/download/shiye-xuiyonghu-oneport-1panel-baota.zip`

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | env PACKAGE_URL=https://github.com/wstimin/shiye-3xuigl-L3/releases/latest/download/shiye-xuiyonghu-oneport-1panel-baota.zip INSTALL_SOURCE=auto bash
```

一键脚本部署成功后，项目由 systemd 服务 `shiye-api` 管理，不需要再到宝塔里创建 Node 项目。服务器上可以随时打开管理菜单：

```bash
shiye
```

菜单标题只显示“管理面板”，可执行安装/更新、查看当前配置、查看状态、重启、日志、配置或取消域名、更新/重装运行文件、数据库迁移、备份和卸载。

也可以提前带入域名和 HTTPS 参数：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | env PACKAGE_URL=https://github.com/wstimin/shiye-3xuigl-L3/releases/latest/download/shiye-xuiyonghu-oneport-1panel-baota.zip INSTALL_SOURCE=auto DOMAIN=panel.example.com ENABLE_NGINX=yes ENABLE_HTTPS=yes CERTBOT_EMAIL=admin@example.com bash
```

继续使用宝塔面板部署时，按下面步骤操作。

注意：一键脚本部署和下面的宝塔手动部署二选一即可。不要一边用一键脚本安装到 `/opt/shiye`，一边又在宝塔里用另一份 `/www/wwwroot/shiye` 创建 Node 项目，否则会出现两个运行目录、两个启动方式，排查会很乱。

进入宝塔：

```text
软件商店
```

确认安装：

```text
Nginx 或 OpenResty
MySQL 或 MariaDB
Node.js 版本管理器，安装 Node.js 20.x
PM2 管理器，可选
```

在宝塔终端确认版本：

```bash
node -v
npm -v
```

Node.js 必须是 20 或更高版本。

## 3. 创建数据库

进入宝塔：

```text
数据库 -> 添加数据库
```

推荐填写：

```text
数据库名：shiye_management
用户名：shiye
密码：自己生成一个强密码
访问权限：本地服务器
字符集：utf8mb4
```

记录数据库连接信息，稍后写入 `.env`：

```text
DATABASE_URL=mysql://shiye:你的数据库密码@127.0.0.1:3306/shiye_management
```

## 4. 上传项目

推荐目录：

```text
/www/wwwroot/shiye
```

上传并解压 Linux 预构建部署包后，目录根部必须能看到：

```text
/www/wwwroot/shiye/package.json
/www/wwwroot/shiye/apps/
/www/wwwroot/shiye/packages/
/www/wwwroot/shiye/dist/
/www/wwwroot/shiye/prisma/
/www/wwwroot/shiye/infra/
/www/wwwroot/shiye/install.sh
```

预构建包里必须已经包含这些运行文件：

```text
/www/wwwroot/shiye/apps/api/dist/main.js
/www/wwwroot/shiye/packages/shared/dist/index.js
/www/wwwroot/shiye/packages/xui-client/dist/index.js
/www/wwwroot/shiye/packages/payment-core/dist/index.js
/www/wwwroot/shiye/dist/user-web/index.html
/www/wwwroot/shiye/dist/admin-web/index.html
```

不要解压成多一层目录，例如：

```text
/www/wwwroot/shiye/shiye-xuiyonghu/package.json
```

如果多了一层，把里面的所有文件移动到 `/www/wwwroot/shiye`。

## 5. 配置环境变量

在宝塔终端执行：

```bash
cd /www/wwwroot/shiye
cp .env.example .env
```

编辑 `.env`，至少修改：

```env
NODE_ENV=production
PORT=3388
PUBLIC_WEB_URL=https://你的域名
DATABASE_URL=mysql://shiye:你的数据库密码@127.0.0.1:3306/shiye_management
SESSION_SECRET=换成强随机字符串
ENCRYPTION_KEY=换成32字节base64密钥
CARD_HASH_SECRET=换成强随机字符串
DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=换成强后台密码
```

生成密钥：

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

`ENCRYPTION_KEY` 必须备份好，丢失后已保存的 3x-ui Token、密码和支付密钥无法解密。

## 6. 安装依赖、初始化数据库

在宝塔终端执行：

```bash
cd /www/wwwroot/shiye
npm ci
npm run install:prod
npm prune --omit=dev
```

注意：不要直接执行 `npm ci --omit=dev` 或 `npm install --omit=dev`。首次安装需要 Prisma、seed 等安装工具。使用预构建包时，`npm run install:prod` 只会检查配置、生成 Prisma Client、执行迁移、初始化管理员和部署检查，不会重新构建 API/管理端/用户端。

如果只是更新数据库和运行文件，仍然推荐执行：

```bash
npm ci
npm run install:prod
npm prune --omit=dev
```

## 7. 创建 Node 项目

进入宝塔：

```text
网站 -> Node 项目
```

推荐填写：

```text
项目名称：shiye-api
项目目录：/www/wwwroot/shiye
启动命令：npm start
Node 版本：20.x
端口：3388
运行用户：www，或宝塔默认用户
```

如果宝塔要求填写环境变量，可以留空，因为项目会读取 `/www/wwwroot/shiye/.env`。

启动后在服务器终端测试：

```bash
curl http://127.0.0.1:3388/api/health
curl http://127.0.0.1:3388/api/setup/status
```

能返回 JSON 即表示 API 正常。

## 8. 配置网站和反向代理

进入宝塔：

```text
网站 -> 添加站点
```

当前版本由 Node.js 一个端口同时提供用户端、管理端和 API：

```text
用户端：https://你的域名/
管理端：https://你的域名/admin/
API：https://你的域名/api/
```

宝塔里只需要把整个网站反向代理到 Node 项目端口，不需要单独配置 `/api/`，也不需要把站点根目录指向 `dist/user-web`。

如果使用宝塔的“反向代理”功能，目标 URL 填：

```text
http://127.0.0.1:3388
```

手动写 Nginx 配置时，使用下面这种整站代理配置，把 `你的域名` 按实际情况替换：

```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://127.0.0.1:3388;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

保存后测试并重载 Nginx。

## 9. 申请 SSL 证书

进入宝塔站点设置：

```text
网站 -> 选择站点 -> SSL
```

申请前确认：

```text
域名已经解析到服务器 IP
宝塔安全和云服务器安全组放行 80/443
http://你的域名 可以正常访问
```

证书成功后，建议开启强制 HTTPS，并把 `.env` 里的 `PUBLIC_WEB_URL` 设置为 `https://你的域名`。

## 10. 支付回调地址

本项目只有用户充值余额业务。用户可输入任意金额，支付成功后增加余额，再用余额续费节点。

支付平台回调地址格式：

```text
https://你的域名/api/payments/epay/notify
https://你的域名/api/payments/bepusdt/notify
https://你的域名/api/payments/alipay/notify
https://你的域名/api/payments/wechat/notify
```

支付结果页：

```text
https://你的域名/payment/result?trade_no=订单号
```

## 11. 更新项目

如果使用一键脚本默认精简部署，运行目录不是 Git 仓库，不能直接 `git pull`。推荐重新执行一键脚本，脚本会保留现有 `.env`，优先获取最新预构建包、迁移数据库、校验并重启服务；如果预构建包不可用，会回退源码构建：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | env PACKAGE_URL=https://github.com/wstimin/shiye-3xuigl-L3/releases/latest/download/shiye-xuiyonghu-oneport-1panel-baota.zip INSTALL_SOURCE=auto bash
```

也可以执行 `shiye`，在“管理面板”菜单里选择：

```text
1. 安装/更新项目
```

只有手动上传完整源码时，才使用下面的源码更新方式。

更新前备份：

```text
MySQL 数据库 shiye_management
/www/wwwroot/shiye/.env
自定义 Nginx 配置
```

更新命令：

```bash
cd /www/wwwroot/shiye
git pull
npm ci
npm run install:prod
npm prune --omit=dev
```

然后在宝塔里重启 Node 项目。

## 12. 卸载项目

如果使用一键脚本部署，推荐执行：

```bash
shiye
```

然后选择：

```text
11. 卸载项目
12. 卸载项目并删除数据库
```

也可以直接使用命令。默认卸载程序、systemd 服务和脚本创建的 Nginx 配置，保留数据库：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/uninstall.sh | bash
```

彻底卸载并删除默认数据库 `shiye_management` 和默认数据库用户 `shiye`：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/uninstall.sh | env DELETE_DATABASE=yes bash
```

如果是宝塔手动部署，需要在宝塔里停止并删除 Node 项目、删除对应站点或反向代理配置，再按需删除实际项目目录和数据库。删除数据库会清空用户、余额、订单、节点、卡密、支付配置和同步日志，操作前务必备份 `.env` 和 MySQL 数据。

## 13. 常见问题

### 启动失败，提示找不到 package.json

项目目录填错，或部署包多解压了一层。确认运行目录下直接存在：

```text
package.json
apps/
packages/
prisma/
```

### API 正常，但页面 404

检查网站是否整站反向代理到 Node.js 服务，例如 `http://127.0.0.1:3388`。当前版本不需要在 Nginx 里直接托管 `dist/user-web`。

### 管理端资源 404

检查反向代理是否保留完整路径并转发到同一个 Node.js 服务。当前版本不需要单独配置 `/admin/assets/`。

### 数据库连接失败

检查 `.env` 中的 `DATABASE_URL`。宝塔本机 MySQL 通常是：

```text
127.0.0.1:3306
```

### 支付回调不到账

检查：

- `.env` 的 `PUBLIC_WEB_URL` 是否是公网 HTTPS 域名
- 支付平台回调地址是否是 `/api/payments/:provider/notify`
- Nginx 是否把整个网站反向代理到 Node.js 服务
- 后台支付通道密钥和签名方式是否正确

## 14. 备份建议

至少备份：

```text
MySQL 数据库 shiye_management
/www/wwwroot/shiye/.env
Nginx 站点配置
```

`.env` 里的 `ENCRYPTION_KEY` 必须保留，否则加密字段无法恢复。
