# 视野 3x-ui 用户管理系统部署教程

本文档对应当前新架构版本：NestJS API、Vue 管理端、Vue 用户端、MySQL、Prisma。生产部署以 `apps/*`、`packages/*`、`prisma/*` 为准，旧版 `server.js` 不作为部署入口。

如果使用宝塔或 1Panel，请优先查看：

- `宝塔部署教程.md`
- `1Panel部署教程.md`
- `UNINSTALL.md`

## 1. 默认部署信息

```text
服务名：shiye-api
安装目录：/opt/shiye
API 端口：3388
用户端入口：https://你的域名/
管理端入口：https://你的域名/admin/
API 前缀：https://你的域名/api/
```

生产环境需要：

- Linux 服务器，推荐 Debian 12、Ubuntu 22.04/24.04、Rocky Linux 9、AlmaLinux 9
- Node.js 20+
- MySQL 8.0+ 或 MariaDB 兼容版本
- Nginx 或 OpenResty
- 一个已解析到服务器的域名，推荐使用 HTTPS

## 2. 一键脚本部署

服务器上直接执行，默认安装到 `/opt/shiye`，服务名 `shiye-api`：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | sudo bash
```

带域名和 HTTPS 直接部署：

```bash
curl -fsSL https://raw.githubusercontent.com/wstimin/shiye-3xuigl-L3/main/install.sh | sudo env DOMAIN=panel.example.com ENABLE_NGINX=yes ENABLE_HTTPS=yes CERTBOT_EMAIL=admin@example.com bash
```

把项目推到 GitHub 后，可以通过 `REPO_URL` 指定仓库地址安装：

```bash
REPO_URL=https://github.com/wstimin/shiye-3xuigl-L3.git \
DOMAIN=panel.example.com \
ENABLE_NGINX=yes \
ENABLE_HTTPS=yes \
CERTBOT_EMAIL=admin@example.com \
bash install.sh
```

如果脚本已经在项目根目录，可以直接执行：

```bash
bash install.sh
```

脚本会完成：

- 检查 root、systemd、包管理器
- 安装基础依赖、Git、Node.js 20+
- 未传入 `DATABASE_URL` 时，安装或使用本机 MySQL/MariaDB
- 创建数据库和数据库用户
- 写入 `/opt/shiye/.env`
- 执行 `npm ci`
- 执行 `npm run install:prod`
- 执行 `npm prune --omit=dev`
- 创建并启动 `shiye-api` systemd 服务
- 可选写入 Nginx 配置和申请 HTTPS 证书

注意：不要直接执行 `npm ci --omit=dev` 或 `npm install --omit=dev` 后再构建。前端和 TypeScript 构建需要 devDependencies。正确顺序是先完整安装依赖，构建完成后再 `npm prune --omit=dev`。

## 3. 使用外部数据库

如果你已经有 MySQL，可以传入完整数据库连接：

```bash
DATABASE_URL='mysql://shiye:强密码@127.0.0.1:3306/shiye_management' \
REPO_URL=https://github.com/wstimin/shiye-3xuigl-L3.git \
DOMAIN=panel.example.com \
ENABLE_NGINX=yes \
ENABLE_HTTPS=yes \
CERTBOT_EMAIL=admin@example.com \
bash install.sh
```

也可以让脚本创建本机数据库：

```bash
MYSQL_DATABASE=shiye_management \
MYSQL_USER=shiye \
MYSQL_PASSWORD='请换成强密码' \
DOMAIN=panel.example.com \
ENABLE_NGINX=yes \
ENABLE_HTTPS=no \
bash install.sh
```

## 4. 手动部署

进入项目目录后创建 `.env`：

```bash
cd /opt/shiye
nano .env
```

示例配置：

```env
NODE_ENV=production
PORT=3388
PUBLIC_WEB_URL=https://panel.example.com
ADMIN_PATH=/admin

DATABASE_URL=mysql://shiye:强密码@127.0.0.1:3306/shiye_management
REDIS_URL=

JWT_SECRET=请换成随机长字符串
SESSION_SECRET=请换成随机长字符串
ENCRYPTION_KEY=请换成32字节以上随机密钥
CARD_HASH_SECRET=请换成随机长字符串

DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=请换成强密码
```

安装、初始化和构建：

```bash
npm ci
npm run install:prod
npm prune --omit=dev
```

启动 API：

```bash
npm start
```

生产环境建议使用 systemd，示例文件在：

```text
infra/systemd/shiye-api.service
```

启用服务：

```bash
cp infra/systemd/shiye-api.service /etc/systemd/system/shiye-api.service
systemctl daemon-reload
systemctl enable --now shiye-api
systemctl status shiye-api
```

## 5. Nginx 配置

示例文件：

```text
infra/nginx/shiye.conf
```

默认结构：

- 用户端静态文件：`/opt/shiye/dist/user-web`
- 管理端静态文件：`/opt/shiye/dist/admin-web`
- API 反向代理：`http://127.0.0.1:3388/api/`

配置后检查并重载：

```bash
nginx -t
systemctl reload nginx
```

## 6. 支付回调地址

支付只用于用户自定义金额充值余额，余额再用于续费。支付平台回调和跳转地址应配置为：

```text
异步回调：https://你的域名/api/payments/支付提供商/notify
同步跳转：https://你的域名/payment/result?trade_no=订单号
```

`.env` 中的 `PUBLIC_WEB_URL` 必须是用户实际访问的公网地址，否则支付跳转、二维码和回调相关链接可能不正确。

## 7. 3x-ui 对接检查

部署完成后，在后台添加 3x-ui 节点并测试连接。当前项目按 `3-xuiAPI文档.txt` 中的接口对接，重点检查：

- 3x-ui 面板地址、认证方式和 Token 是否正确
- 节点列表和用户同步是否正常
- 续费后是否能同步到 3x-ui
- 流量重置接口是否与当前 3x-ui 版本一致

不同 3x-ui 版本或修改版可能存在接口差异，部署后请先用测试用户验证同步和续费流程。

## 8. 更新项目

```bash
cd /opt/shiye
git pull
npm ci
npm run install:prod
npm prune --omit=dev
systemctl restart shiye-api
```

更新后检查：

```bash
systemctl status shiye-api
journalctl -u shiye-api -n 100 --no-pager
```

## 9. 备份

至少备份：

```text
/opt/shiye/.env
MySQL 数据库 shiye_management
Nginx 或 OpenResty 站点配置
```

备份数据库：

```bash
mysqldump -u shiye -p shiye_management > shiye_management.sql
```

`.env` 里的 `ENCRYPTION_KEY` 必须保存。丢失后，已经加密保存的 3x-ui Token、密码和支付密钥无法解密。

## 10. 常用排查命令

查看服务：

```bash
systemctl status shiye-api
journalctl -u shiye-api -f
```

检查端口：

```bash
ss -lntp | grep 3388
```

检查 Nginx：

```bash
nginx -t
systemctl status nginx
```

检查数据库连接：

```bash
mysql -u shiye -p -h 127.0.0.1 -P 3306 shiye_management
```

重新执行生产初始化：

```bash
cd /opt/shiye
npm ci
npm run install:prod
npm prune --omit=dev
systemctl restart shiye-api
```

## 11. 卸载

卸载和删除命令已经单独整理在：

```text
UNINSTALL.md
```

只重装程序时不要删除数据库。确认不再需要业务数据后，再执行删除数据库的命令。
